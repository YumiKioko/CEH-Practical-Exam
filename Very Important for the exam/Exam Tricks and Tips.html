Exam-Specific Tricks
Proven strategies for common exam scenarios

Network/Scanning Questions:

# ALWAYS start with this:
nmap -sV -sC -p-  -oN scan.txt

# While that runs, scan common ports in another terminal:
nmap -sV -sC -p 21,22,23,25,80,110,139,143,443,445,3306,3389,8080 

# Quick service enumeration while waiting:
nmap -sU -p 53,67,68,161,162  &  # UDP ports in background

Web Applications Attack Order:

1. gobuster/dirb first
2. Look for /admin, /login, /upload, /config, /backup
3. Try basic SQLi: ' OR '1'='1
4. Check source code (Ctrl+U)
5. Use Burp Suite for complex requests
6. Test for LFI: ../../../../etc/passwd

# Quick web assessment combo:
gobuster dir -u http://target -w common.txt -x php,txt,html &
nikto -h http://target &
whatweb http://target

Windows Targets:

# If you find SMB (445):
1. enum4linux -a 
2. smbclient -L // -N
3. crackmapexec smb  -u '' -p ''
4. If you see MS17-010, use EternalBlue
5. Check for anonymous access: smbclient ///share -N

# Additional Windows checks:
rpcclient -U "" -N   # Check null session
nbtscan              # NetBIOS information

Common Mistakes to AVOID
Critical do's and don'ts for exam success

‚ùå DON'T:

Use wrong IP (confusing target with attacker)
Forget to set your IP as LHOST in payloads
Fail to verify if service is the vulnerable version
Spend 2 hours on ONE difficult question
Forget to activate monitor mode in WiFi
Ignore the answer format (flag format)
Overlook default credentials
Skip basic manual testing for automated tools only

‚úÖ DO:

ALWAYS verify your IPs before executing
Read flags carefully (some are case-sensitive)
Try default credentials (admin/admin, root/toor, guest/guest)
If something doesn't work in 15 min, move to another question
Use tab-completion and history (up arrow)
Copy/paste commands from cheat sheet to avoid typos
Take screenshots of important findings
Check both HTTP and HTTPS for web services

Quick Reference Commands
Essential commands for rapid execution

Essential Nmap Scans:

# Comprehensive TCP scan
nmap -sS -sV -sC -O -p-  -oA full_scan

# Quick top ports
nmap --top-ports 100 

# UDP important ports
nmap -sU -p 53,67,68,69,123,161,162,514 

# Vulnerability scan
nmap --script vuln

Web Testing Quick Commands:

# Directory brute force
gobuster dir -u http://target -w common.txt -x php,html,txt

# Subdomain enumeration
gobuster dns -d domain.com -w subdomains.txt

# Nikto scan
nikto -h http://target -o nikto_scan.txt

# Check for WAF
wafw00f http://target

Privilege Escalation Quick Checks:

# Linux quick checks

# Windows quick checks
