1. HashCalc

Purpose: Calculate and verify hash values for files and text strings.

Supported Hash Algorithms

MD5
128-bit hash (deprecated)
SHA-1
160-bit hash (deprecated)
SHA-256
256-bit hash (recommended)
SHA-512
512-bit hash (highly secure)

Key Features

  • File hash calculation with drag-and-drop support
  • Text string hashing for passwords and data
  • Hash comparison and verification
  • Clipboard integration for quick workflows
  • Multiple hash algorithms simultaneously

Common Use Cases

  • Verify file integrity after downloads
  • Check software authenticity
  • Compare file versions
  • Generate checksums for backups

2. VeraCrypt

Purpose: Open-source disk encryption software (successor to TrueCrypt).

Core Features

  • Encrypted Containers: Virtual encrypted disks stored as files
  • Full Disk Encryption: Encrypt entire system drives
  • Partition Encryption: Protect specific disk partitions
  • Hidden Volumes: Plausible deniability with secret volumes

Encryption Algorithms

Algorithm Key Size Security Level
AES 256-bit High
Serpent 256-bit High
Twofish 256-bit High
AES-Twofish-Serpent 768-bit cascade Maximum

Basic Workflow

# Create encrypted container 1. VeraCrypt ‚Üí Create Volume 2. Select "Create an encrypted file container" 3. Choose volume type (Standard or Hidden) 4. Select file location and size 5. Choose encryption algorithm 6. Set password (20+ characters recommended) 7. Format volume # Mount volume 1. VeraCrypt ‚Üí Select File 2. Click Mount 3. Enter password 4. Access mounted drive # Dismount Select mounted volume ‚Üí Click Dismount
Best Practices:
  • Use passwords with 20+ characters
  • Enable keyfiles for two-factor authentication
  • Regular backups of volume headers
  • Always dismount volumes when not in use
  • Use PIM for additional iterations

3. Cracking Hashes

Purpose: Password recovery and security testing by reversing hash values.
‚ö†Ô∏è Ethical Notice: Only crack hashes you own or have explicit permission to test. Unauthorized access is illegal.

Hashcat - GPU Accelerated

# Basic syntax hashcat -m [hash_type] -a [attack_mode] [hash_file] [wordlist] # MD5 dictionary attack hashcat -m 0 -a 0 hashes.txt rockyou.txt # SHA-256 brute force (6 characters) hashcat -m 1400 -a 3 hash.txt ?a?a?a?a?a?a # Common hash types -m 0 MD5 -m 100 SHA1 -m 1400 SHA256 -m 1800 SHA512 -m 3200 bcrypt -m 1000 NTLM (Windows) # Attack modes -a 0 Dictionary attack -a 3 Brute-force -a 6 Hybrid (wordlist + mask) -a 7 Hybrid (mask + wordlist)

John the Ripper

# Basic usage with wordlist john --wordlist=rockyou.txt hashes.txt # Show cracked passwords john --show hashes.txt # Brute force mode john --incremental hashes.txt # Specify hash format john --format=raw-md5 hashes.txt john --format=bcrypt hashes.txt # Use rules for mutations john --wordlist=passwords.txt --rules hashes.txt

Attack Strategies

Dictionary Attack

Uses wordlists to match hashes. Fast and effective for common passwords.

Brute Force

Tries all possible combinations. Slow but guaranteed if enough time.

Hybrid Attack

Combines wordlists with rules and mutations for variations.

Rainbow Tables

Pre-computed hash lookups. Fast but requires storage space.

Hash Identification

# Using hashid hashid -m [hash_value] # Example hashid -m 5f4dcc3b5aa765d61d8327deb882cf99 # Outputs: MD5 (-m 0 for hashcat)

4. BCTextEncoder

Purpose: Text encryption and encoding tool with multiple transformation methods.

Supported Methods

  • Base64: Binary-to-text encoding
  • Hex: Hexadecimal representation
  • URL Encoding: Percent-encoding for URLs
  • Binary: Binary string conversion
  • ROT13: Simple letter substitution
  • Various ciphers: Caesar, Vigen√®re, etc.

Basic Workflow

# Encoding Process 1. Enter plaintext in input field 2. Select encoding method (Base64, Hex, etc.) 3. Click "Encode" button 4. Copy encoded result # Decoding Process 1. Paste encoded text 2. Select same encoding method 3. Click "Decode" button 4. View original plaintext
‚ö†Ô∏è Security Note: Basic encoding (Base64, Hex, URL) is NOT encryption. These methods are easily reversible and provide no security. Use proper encryption for sensitive data.

Use Cases

  • Obfuscate configuration data (not security-critical)
  • Quick text transformations for development
  • Data format conversions
  • Educational purposes for learning encoding schemes

Purpose: Comprehensive educational platform for cryptography and cryptanalysis.

CrypTool 2 (CT2) Windows Visual programming, workflow-based

Classical Ciphers

  • Caesar Cipher: Simple shift cipher

  • Playfair Cipher: Digraph substitution
  • Transposition: Rearrangement of letters
  • Enigma Machine: Historical rotor machine simulator

Modern Cryptography

Symmetric
AES, DES, 3DES, Blowfish
Asymmetric
RSA, ECC, ElGamal
Hash Functions
MD5, SHA-1, SHA-256, SHA-3
Digital Signatures

Cryptanalysis Tools

Learning Path

  • Start with Caesar cipher and frequency analysis
  • Learn hash functions (MD5, SHA-256)
  • Explore symmetric encryption (AES)
Intermediate Level:
  • Study public key cryptography (RSA)
  • Understand digital signatures
  • Practice key exchange protocols

  • Advanced cryptanalysis techniques

6. Security Best Practices

Password Security

  • Use 16+ characters with mixed complexity
  • Never reuse passwords across services
  • Use a password manager (KeePass, Bitwarden)
  • Enable multi-factor authentication everywhere
  • Avoid dictionary words and personal information

Encryption Guidelines

  • Avoid deprecated algorithms: MD5, SHA-1, DES, RC4
  • Use modern standards: AES-256, SHA-256, RSA-4096
  • For passwords: Use bcrypt, scrypt, or Argon2
  • Key management: Store keys separately from encrypted data

Recommended Hash Functions

Purpose Recommended Avoid
File Integrity SHA-256, SHA-3 MD5, SHA-1
Password Storage bcrypt, Argon2 Plain SHA, MD5
Digital Signatures SHA-256, SHA-512 SHA-1

Ethical Considerations

  • Only test systems you own or have written permission to test
  • Use encryption to protect sensitive data, not hide malicious activity
  • Follow responsible disclosure for security vulnerabilities
  • Comply with local laws regarding cryptography and computer security
  • Respect privacy and confidentiality
