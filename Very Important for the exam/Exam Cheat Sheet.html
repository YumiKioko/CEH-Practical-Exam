Steganography & Data Hiding

Hidden data detection and extraction techniques

Tool Selection Guide:

  • Steghide: JPG, BMP, WAV, AU files
  • OpenStego: PNG files (primary format)
  • Snow: Text files (TXT) - whitespace steganography

Steghide - For JPG, BMP, WAV, AU Files

# ✅ Supported: JPEG, BMP, WAV, AU

# Hide data
steghide embed -cf image.jpg -ef secret.txt
steghide embed -cf image.bmp -ef data.txt -p password
steghide embed -cf audio.wav -ef message.txt -p mypassword

# Extract hidden data
steghide extract -sf image.jpg
steghide extract -sf image.bmp -p password
steghide info image.jpg

# Crack password
stegseek image.jpg rockyou.txt

Steghide Key Points:

  • Uses compression and encryption
  • Supports password protection
  • ⚠️ Does NOT support PNG or GIF formats
  • Best for: JPG/BMP images and WAV/AU audio

OpenStego - For PNG Files

# ✅ Primary format: PNG

# Hide data in PNG
openstego embed -mf secret.txt -cf cover.png -sf output.png -p password
openstego embed -mf message.txt -cf image.png -sf stego.png

# Extract hidden data
openstego extract -sf stego.png -xd /output/directory -p password
openstego extract -sf output.png -xf extracted.txt

# GUI mode
openstego

OpenStego Key Points:

  • Optimized specifically for PNG format
  • Uses LSB (Least Significant Bit) algorithm
  • User-friendly GUI interface available
  • Best for: PNG image steganography

Snow - For Text Files (TXT)

# ✅ Format: Plain text files (TXT, ASCII)
# Hides data in whitespace (spaces/tabs)

# Hide message in text file
snow -C -m "secret message" -p password input.txt output.txt
snow -C -m "hidden data" input.txt output.txt

# Extract hidden message
snow -C -p password output.txt
snow -C stego.txt

Snow Key Points:

  • Whitespace steganography - uses spaces and tabs
  • Hidden data invisible to casual observers
  • Works with any ASCII text file
  • Best for: Source code, plain text documents

Quick Reference Table

Tool File Types Embed Command Extract Command
Steghide JPG, BMP, WAV, AU steghide embed -cf image.jpg -ef secret.txt steghide extract -sf image.jpg
OpenStego PNG openstego embed -mf secret.txt -cf image.png -sf output.png openstego extract -sf output.png -xf extracted.txt
Snow TXT (text files) snow -C -m "message" -p pass input.txt output.txt snow -C -p pass output.txt

⚠️ Important Notes:

  • Steghide: Cannot handle PNG or GIF - use for JPG/BMP/WAV/AU only
  • OpenStego: Specifically designed for PNG files
  • Snow: Only for text files - data lost if reformatted
  • Always test extraction immediately after embedding
  • Password-protected data cannot be recovered without password

✅ CEH Exam Tips - Steganography:

  • First identify file type: file suspicious.file
  • JPG/BMP/WAV/AU → Use Steghide
  • PNG → Use OpenStego
  • TXT → Use Snow
  • If password protected, try common passwords or stegseek
  • Check metadata: exiftool file

CEH Exam Tips

CEH Practical Exam Tips:

  • Time Management: 6 hours for 20 questions - spend ~15-18 minutes per challenge
  • Read Carefully: Questions often contain hints about the vulnerability type
  • Document Everything: Keep notes of IPs, credentials, and findings
  • Start with Reconnaissance: Always run nmap first to identify services
  • Check for Low-Hanging Fruit: Default credentials, anonymous access, misconfigurations
  • Use Multiple Tools: If one tool fails, try alternatives
  • Follow the Methodology: Reconnaissance → Scanning → Enumeration → Exploitation

Common Exam Scenarios:

  • Web application vulnerabilities (SQLi, XSS, LFI/RFI)
  • Network service exploitation (SMB, FTP, SSH)
  • Password cracking and hash analysis
  • Privilege escalation (Windows & Linux)
  • Steganography and data hiding
  • Traffic analysis with Wireshark
  • Cryptography challenges

Common Mistakes to Avoid:

  • Not reading the question completely
  • Skipping enumeration and jumping straight to exploitation
  • Using wrong IP addresses (target vs. attacker)
  • Not checking for multiple vulnerabilities
  • Forgetting to escalate privileges after initial access
  • Not trying default credentials first
  • Spending too much time on one question
# Quick Methodology Reminder
1. Reconnaissance: nmap, service detection
2. Enumeration: Detailed service analysis
3. Vulnerability Scanning: searchsploit, CVE lookup
4. Exploitation: Gain initial access
5. Post-Exploitation: Privilege escalation, data collection
6. Reporting: Document findings and steps
